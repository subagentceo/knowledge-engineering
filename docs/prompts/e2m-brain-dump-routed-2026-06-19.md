## 7. schema — the 3 record kinds (pydantic + zod mirror, from envelope.ts)

reproduced for cross-language reviewers; envelope.ts stays the source. zod already exists generated (envelope.zod.ts); pydantic is shown as the target of open/next #1.

```python
# pydantic v2 — target of: datamodel-code-generator envelope.schema.json
from typing import Literal, Optional
from pydantic import BaseModel, Field

EnvelopeType    = Literal["task", "ack", "result", "escalate", "notify", "summary", "operator"]
EnvelopeState   = Literal["pending", "read", "actioned", "archived"]
TaskState       = Literal["pending", "in_progress", "blocked", "completed", "failed"]
TransitionEvent = Literal["claim", "complete", "block", "unblock", "fail", "retry", "ack", "read"]

class Envelope(BaseModel):
    type_: Literal["envelope"] = Field(alias="_type")
    id: str
    envelope_type: EnvelopeType
    from_: str = Field(alias="from")
    to: str
    subject: str
    at: str
    state: EnvelopeState
    priority: Optional[Literal[1, 2, 3, 4, 5]] = None
    thread_id: Optional[str] = None
    reply_to: Optional[str] = None
    payload: Optional[dict] = None
    requires_decision: Optional[bool] = None
    decision_options: Optional[list[str]] = None
    ack_required: Optional[bool] = None
    expires_at: Optional[str] = None

class DurableTask(BaseModel):
    type_: Literal["task"] = Field(alias="_type")
    id: str
    queue: str                               # NOT "domain" — that key is invalid
    subject: str
    state: TaskState
    created_at: str
    updated_at: str
    from_: Optional[str] = Field(default=None, alias="from")
    ke_fit_score: Optional[Literal[1, 2, 3, 4, 5]] = None
    owner: Optional[str] = None
    depends_on: Optional[list[str]] = None
    blocks: Optional[list[str]] = None
    payload: Optional[dict] = None
    result: Optional[dict] = None
    error: Optional[str] = None

class Transition(BaseModel):
    type_: Literal["transition"] = Field(alias="_type")
    id: str                                  # id of the task/envelope being transitioned
    at: str
    event: Optional[TransitionEvent] = None
    prior_state: Optional[str] = None
    new_state: Optional[str] = None
    owner: Optional[str] = None
    status: Optional[str] = None
    acked_by: Optional[str] = None
    read_by: Optional[str] = None
    note: Optional[str] = None
    error: Optional[str] = None
```

```typescript

  _type: z.literal("envelope"),
  id: z.string(),
  envelope_type: z.enum(["task", "ack", "result", "escalate", "notify", "summary", "operator"]),
  from: z.string(), to: z.string(), subject: z.string(), at: z.string(),
  state: z.enum(["pending", "read", "actioned", "archived"]),
  priority: z.union([z.literal(1), z.literal(2), z.literal(3), z.literal(4), z.literal(5)]).optional(),
  thread_id: z.string().optional(), reply_to: z.string().optional(),
  payload: z.record(z.string(), z.unknown()).optional(),
  requires_decision: z.boolean().optional(), decision_options: z.array(z.string()).optional(),
  ack_required: z.boolean().optional(), expires_at: z.string().optional(),
});
```

## directives

<emit_typed_records_never_raw_json>
every line appended to cowork/data/mailbox/*.jsonl or cowork/data/queues/*.jsonl MUST validate as exactly one of envelope|task|transition. coworkers never hand another coworker a raw blob (see envelope.ts, mailbox-envelope-canon.md).
</emit_typed_records_never_raw_json>

<single_source_then_generate>
edit envelope.ts only. regenerate bindings with `npm run schema:gen`; never hand-author zod/pydantic/serde in parallel (refs: t2z).
</single_source_then_generate>

<append_only_latest_line_wins>
storage is append-only JSONL. for a given id, the last line is current state. concurrent coworkers interleave lines, never overwrite — this is the no-clobbering guarantee.
</append_only_latest_line_wins>

<keep_it_stupid_simple>
no broker, no daemon, no database in the hot path. the records are the contract; storage is swappable (project into Postgres/redis later, refs: pgd). do not add infrastructure the dogfooding has not demanded.
</keep_it_stupid_simple>

<borrow_dont_reinvent>
prefer existing upstream primitives: ext-tasks for the task state-machine (refs: ext), CommonMark for specs (refs: cm), CSL for citations (refs: csl), the redis triplet for cache parity. write new code only where the monolith has a real gap.
</borrow_dont_reinvent>

<three_first_class_bindings>
TypeScript, Python, and Rust are co-equal e2m bindings generated from envelope.ts — never privilege one over the others. envelope.py (pydantic) and envelope.rs (serde) ship together; the chain is incomplete until all three validate the same envelope.schema.json (refs: t2z). every upstream the protocol touches — modelcontextprotocol, a2aproject, agentclientprotocol, redis, anthropics — already ships all three (refs: mot, aat, agt, ret, ant).
</three_first_class_bindings>

<vendor_neutral_base_images>
pin every runtime to the ubuntu Docker org — rust (refs urs), python (refs upy), node (refs und), postgres (refs ups), redis (refs urd). prefer these vendor-neutral images over single-vendor base images for trust and reproducibility (refs: ubu).
</vendor_neutral_base_images>

<oauth_subscription_is_the_real_backend>
the paid claude.ai OAuth subscription is the canonical real-Claude path — better value than provisioning an ANTHROPIC_API_KEY. the only ANTHROPIC_API_KEY that may exist is the local sentinel (ollama / lm-studio / llama-cpp / local), confined to the Tailscale-routed dev shell; never a real key, never in the cloud chassis (refs: kdn, ke).
</oauth_subscription_is_the_real_backend>

<fetch_coworker_optimized_md_not_html>
crawl jobs fetch the coworker-optimized `.md` article (append .md + Accept: text/markdown), never the operator HTML — no turndown, fewer tokens, deterministic. collection pages have no .md, so generate _collection.md per crawl. only /en/ content (refs: scs).
</fetch_coworker_optimized_md_not_html>

<cache_fixed_size_not_ttl>
all crawl/token caches are capacity-bounded LRU — L1 lru-cache max, L2 redis maxmemory allkeys-lru, L3 postgres durable — never TTL-based. drop the 7-day redis object TTL for these keys; rely on allkeys-lru eviction so re-crawls hit cache and save tokens (refs: lru).
</cache_fixed_size_not_ttl>

<reuse_repo_crawl_primitives>
support.claude.com crawling reuses scripts/crawl-vendors.ts (support-mdfirst per-host override), url-to-path topology, sitemap-xml parsing, and src/cache/tiered.ts. a new vendor only needs crawl.json + collections.yaml; never a second crawler (refs: che).
</reuse_repo_crawl_primitives>
