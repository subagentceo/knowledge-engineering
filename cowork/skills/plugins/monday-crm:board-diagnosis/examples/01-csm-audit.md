## What this example demonstrates

- All five diagnosis strands run; results explicitly scored.
- Automation probe via `all_monday_api` (no wrapped tool for automation list).
- P0/P1/P2 prioritization tied to forecast/reporting impact, not just fill-rate.
- Review-each Proactive mode → batched review doc (cheaper than 9 individual confirms).
- Safety rail held: no group deletes, no column archives without user action, no stage/amount edits.
- Doc body carries `Generated by Claude` footer + `<!-- claude-skill-id: board-diagnosis -->` comment for findability + same-day idempotency; fix-task items carry `Source = Claude` on a status column.
