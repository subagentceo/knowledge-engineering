metrics cadence every wednesday · 10:00 utc · data-driven pm loop
approach: metrics-review skill (customized) runs against dw.fact_connectors + dw.dim_agent_templates on a scheduled task (wednesday 10:00 UTC). a collector agent snapshots tool_calls, approval_rate, and ke_fit_score. an analyst agent generates the scorecard HTML. a pm agent writes the roadmap delta. output lands in cowork/artifacts/ as a dated html file.

multi-agent product-management focus: cowork/ · @cite C2 · @cite vendor/anthropics/platform.claude.com/…/managed-agents
recommendation: run three specialized agents in parallel each wednesday. the collector has readonly access to alloydb. the analyst uses the metrics-review skill + data:analyze + data:build-dashboard. the pm agent uses product-management:roadmap-update + product-management:write-spec to produce a delta spec. all three write back to cowork/artifacts/. no single agent needs full repo access.
collector
Snapshots dw.fact_connectors, dw.dim_agent_templates, and dw.dim_desktop into a JSONL report for the current week. Runs first; downstream agents depend on its output.

analyst
Ingests collector JSONL. Runs metrics-review skill to produce scorecard: tool_calls trend, approval_rate by connector, ke_fit_score distribution across templates. Emits a dated HTML artifact.

north star cowork/ product vision · incremental progress
vision: cowork/ is the operating system for this repo — a live, data-driven set of HTML pages that makes the repo's state observable, its metrics reviewable on a schedule, and its product direction evolvable by multi-agent collaboration. The html-effectiveness repo (C3) is the pattern library. The agent-native templates (C4–C15) are the building blocks. The Kimball DW (fact_connectors, dim_desktop, dim_agent_templates) is the data substrate. The skill inventory (38 skills across 4 domains) is the agent capability surface.

incremental progress path: install analytics (C4) + plan (C5) → wire fact_connectors to analytics → activate wednesday 10:00 UTC review → add brain (C6) for knowledge recall → add dispatch (C13) for agent orchestration.
