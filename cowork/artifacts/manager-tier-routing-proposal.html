Flow: operator.jsonl intake → classify (haiku, scores 12 domains) → route envelope (envelope_type: "task", to: <manager>). On ambiguity the classifier emits to: project-manager. Every hop is one typed JSONL line — fully replayable.

What a manager is (manager : coworker :: lead : teammate)

A manager is a coworker with a supervisory lifecycle. It does not do the atomic work — it decomposes, dispatches, aggregates:

INTAKE → CLASSIFY → DECOMPOSE → DISPATCH(coworker) → AGGREGATE → REPORT(up)

New manifest fields: tier: manager, manages: [<domain>-coworker], reports_to: operator. project-manager additionally carries role: router — the only one that reads operator.jsonl intake.

Refinement of your request (the "modify" part)

Don't hand-write 12 near-identical skills. That's the drift trap we already hit twice (the Fivetran format, the 8-token system). Instead:

  • One <manager_lifecycle> contract + one managers.manifest.json (id, domain, manages, peers).
  • Generate the 12 thin SKILL.md from that manifest — they differ only by domain/queue/managed-coworker.
  • project-manager = the manager lifecycle + a router module (the classifier). One special case, not twelve.
