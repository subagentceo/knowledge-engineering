# Claude in Chrome — The Bridge for Tools with No Connector

For anything that lives in a browser, Claude can read, click, and navigate pages. It and Cowork work together in **one conversation, both surfaces**: Claude gathers info / takes actions in the browser, then builds the deliverable back in Cowork.

What it unlocks:
- **Internal dashboards** (Tableau, Looker, BI tools) — pull numbers, download them, use as context for Cowork tasks.
- **Vendor portals and customer systems** behind SSO with no API — navigate, pull, act.
- **Web apps behind a login** — even tools your team built become scriptable: *"Open the procurement system, find every PO from our top ten suppliers in Q3, pull the line items into a spreadsheet."*
- **Web research that ends in a deliverable** — open ten tabs, pull each, turn it into a brief without copy-paste.

The pattern: any time you think *"I'd love to give this to Claude, but it lives on the web,"* Claude in Chrome is the answer.

Control and watch-outs: by default it **asks before sensitive actions** — approve or deny each. **You must be signed in** — Claude can't log in for you; it works in the session you've already authenticated. **Be deliberate about access** — on the open web Claude sees what you see, including anything you have access to; for sensitive sites narrow what it can act on and review actions before approving.
