# Claude in Microsoft 365 — Working Inside the Document

Claude appears as an **add-in inside Word, Excel, PowerPoint, and Outlook**, working on the files you have open. One conversation can carry context across the apps.

- **Excel** — analyze data, write formulas, debug `#REF!`/circular references, run scenario tests, build from a template, cite specific cells. *"Pull actuals from Q3, compare to the Q3 plan in the same workbook, write variance commentary in column F."*
- **PowerPoint** — read your slide master/fonts/colors and build matching slides; generate native editable charts (not pasted images); operate on the selected slide. *"Turn the Excel analysis into a three-slide QBR deck using our template."*
- **Word** — draft/revise/reformat in place, work with comments and tracked changes, ground drafts from connected sources.
- **Outlook** — triage mail with context from your other work; draft replies reflecting prior threads, calendar, and recent decisions.

Cross-app moves are the differentiator (Outlook → Word → Excel → PowerPoint → back to Outlook) — you carry one document's context into the next.

## Cowork vs. Claude in M365 — when to use which

- **Cowork** — when the work pulls from **many sources and ends in a deliverable** (build a brief from twenty files; assemble a report from Salesforce + Slack; run a workflow on a schedule).
- **Claude in M365** — when you're **in the Office file itself**, editing in place and carrying context app to app.

Most real work uses both, and they hand off cleanly — the file is the file in both cases.
