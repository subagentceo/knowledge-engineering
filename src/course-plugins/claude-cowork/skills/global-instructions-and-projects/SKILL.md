# Standing Context: Global Instructions and Projects

> Distilled from the *Introduction to Claude Cowork* course.

## Overview

In Chat, memory builds itself. **Cowork is different — the context that carries task to task is mostly the context you set up.** Two layers: global instructions (apply to every session) and projects (Claude builds memory automatically inside them). Most people start with global instructions and add projects as recurring work surfaces.

- [Global instructions](references/global-instructions.md) — the standing brief written once in Settings, what to put in it, and how corrections you keep giving become candidates for it.
- [Projects](references/projects.md) — a scoped workspace for a stream of work, the four parts (instructions, scheduled tasks, context, and the memory Claude builds), good candidates, and the three ways to start one.
