# Projects — A Scoped Workspace for a Stream of Work

Global instructions cover *you*; projects cover *what you're working on*. A project is tied to a specific stream — a customer, a recurring deliverable, a launch. Inside are three things you set up and one Claude builds:

- **Instructions** — like global, but scoped to this project.
- **Scheduled tasks** — recurring runs that belong to the project and run with its context.
- **Context** — one or more folders or links every conversation in the project can access.
- **Memory** — what Claude learns from conversations inside the project. **This builds over time; you don't write it.**

Memory is the difference projects make: outside a project each session starts fresh (apart from global instructions); inside one, every conversation adds to what Claude knows — so the next task opens with the client's situation, last week's decisions, and what's still open already in hand.

Good candidates for a project: a **client/account**, a **recurring deliverable** (monthly report, weekly update), a **launch or initiative**.

## Three ways to start a project

Click **Projects** in the Cowork sidebar → **New project**, then choose:
- **From scratch** — start empty, add instructions/context as you go.
- **From an existing folder** — it becomes the project's working directory.
- **From a Chat project** — transition instructions and knowledge over. **One-way only** — changes in Cowork don't sync back to Chat.

You can change the working folder, instructions, and connectors at any time.
