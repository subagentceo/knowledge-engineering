# Operator prompt — 2026-05-10

The operator-stated directives that authored PR #3 (this branch).
Treat as a **starting point, not a spec.** When a directive here conflicts
with a cited Anthropic / Claude Code / MCP / vendor doc, the cited doc wins
and the conflict is logged in the PR description.

## Auth and scope

- **OAuth only.** Use the existing OAuth gate (`src/oauth/token.ts` →
  `CLAUDE_CODE_OAUTH_TOKEN`). `ANTHROPIC_API_KEY` must never be set; the gate
  fails closed when it is.
- **No Managed Agents API calls.** We **do** ingest the Managed Agents
  documentation and reuse its rubric/outcome pattern via the Messages API
  over OAuth, but we do not build managed-agent products and do not call
  the managed-agents endpoints.

## Working discipline

- **Commit-per-todo / per-subtask / per-task.** Each `TodoWrite` transition
  `in_progress → completed` triggers a commit. Each subtask completion is
  a commit. Each task completion is a commit. Atomic blast radius;
  PR diff readable per stride.
- **No one-shotting.** Multi-turn agent loop with prompt caching (existing
  `src/examples/{prompt-caching,tool-caching}.ts`) and sub-agent task
  assignments. Each sub-agent call has a measurable outcome before context
  is compacted.
- **ODD + TDD + Citations.** Every new test file ships with a `@cite`
  header referencing source docs. CI guard `verify:citations` parses test
  files and rejects any test missing a `@cite` header that resolves to a
  real path under `vendor/`.
- **Operator prompts are seeds.** When a directive in this prompt conflicts
  with a cited doc, the cited doc wins and the conflict is logged in the PR
  description.

## Slash command surface

A single user-facing umbrella `/routines` at `.claude/skills/routines.md`.
`/routines run "<expr>"` dispatches to `/loop` (interval) or `/schedule`
(cron-style recurrence). Existing `.claude/loop.md` and
`.claude/skills/schedule-bridge.md` are kept; `routines.md` is the
user-facing surface that emits one or the other.

## ODD / rubrics

Every phase from Phase 0 onward ships `rubrics/phase-<N>.md` modeled on
`define-outcomes.md`. Grading is mechanical (`scripts/grade-phase.ts`,
Phase 9) — **not** the Managed Agents API. Messages API over OAuth with
fresh context per criterion (per `define-outcomes.md`'s "separate context
window to avoid being influenced").

## Long-arc goal

Per the publicly-documented Boris Cherny pattern (Anthropic), a sufficiently
scaffolded agent system can write 100% of a codebase. PR 3 (this branch)
ships Phase 0: posture + seeds + citations + rubric stubs + the
Full-Stack Cloud Agent runner scaffolding. Future PRs relocate the
orchestrator into a Cloudflare Worker (Durable Object) that wakes on
Cron Triggers; vendor mirror hydrates from R2; the agent writes 100% of
new code by reading rubrics and opening PRs against itself.
